# MSportFit — правила проєкту (Cursor)

Monorepo: `/client` + `/server`. Цей документ — **обов'язкові** правила для Cursor та розробки.

---

## 1. Стек (фіксований, без змін без явного запиту)

| Частина | Технології |
|--------|-------------|
| Frontend | React, Vite, React Router, SCSS |
| State | Redux Toolkit **тільки** для: auth, favorites, products, фільтри списку вправ |
| UI-допомога | React Select |
| Backend | Node.js, Express |
| БД | PostgreSQL (Docker Compose) |
| ORM | Prisma |
| Auth | bcrypt + JWT (Bearer) |

**Заборонено:** додавати нові бібліотеки/фреймворки без явного запиту користувача.

---

## 2. Дані

- **PostgreSQL (Prisma):** таблиці `users`, `products`, `favorites`.
- **Frontend (локальні JSON):** `exercises.json`, `faq.json`, опційно `programs.json`.
- **Відео:** тільки YouTube embed (без власного хостингу відео).

---

## 3. Принципи розробки

- **KISS + SOLID** — простий, зрозумілий код без надмірної абстракції.
- **Без оверінжинірингу:** не вводити патерни/шари “на майбутнє”, якщо це не потрібно зараз.
- **1 промпт = 1 невеликий результат:** одна логічна зміна за раз (одна фіча, один фікс, один файл/модуль).
- **Неясність:** якщо вимоги або контекст неоднозначні — **обов’язково запитати** перед припущеннями. Не домислювати.

---

## 4. Обмеження змін

- **Заборонено без явного запиту:**
  - великі рефакторинги;
  - масові перейменування (файлів, змінних, API);
  - додавання нових залежностей у `package.json`.
- Зміни — **точечні**, мінімально необхідні для виконання поточного завдання.

---

## 5. Вихід після КОЖНОГО промпту (обов’язковий формат)

Після виконання завдання Cursor **обов’язково** виводить:

1. **Змінені файли** — список шляхів (наприклад: `client/src/pages/Login.jsx`, `server/src/routes/auth.js`).
2. **Що зроблено** — короткі буллети (що саме додано/змінено/видалено).
3. **Чеклист перевірки** — як запустити та що перевірити (команди, кроки, очікувана поведінка).

*Приклад:*
- Змінені файли: `client/src/store/slices/authSlice.js`
- Зроблено: додано збереження `token` у `localStorage` при логіні; додано очищення при logout.
- Перевірка: `npm run dev` у `client`, логін → перезавантажити сторінку → має залишатися авторизація.

---

## 6. Git-дисципліна

- **Часті невеликі коміти** — один логічний крок на коміт.
- **Змістовні повідомлення** — що змінено і навіщо (укр/англ на вибір проєкту).
- **Регулярний push** — не накопичувати тільки локальні коміти.
- **Гілки за задачами** — для окремих фіч/фіксів краще окрема гілка, потім merge у основну.

---

## 7. Підсумок для Cursor

| Правило | Дія |
|--------|-----|
| Неясність у завданні | Запитати, не припускати |
| Один промпт | Одна невелика ітерація |
| Нові бібліотеки / великі рефактори / масові перейменування | Тільки за явним запитом |
| Після виконання | Завжди: змінені файли + що зроблено + чеклист перевірки |
| Код | KISS + SOLID, без оверінжинірингу |
